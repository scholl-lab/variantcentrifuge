<objective>
Pre-parse GT column once at DataFrame load time and eliminate redundant regex parsing across downstream stages.

Purpose: GT column parsing (regex `([^()]+)\(([^)]+)\)`) currently happens independently in converter.py (finalize_excel_file for IGV links), converter.py (produce_report_json for IGV JSON), and generate_igv_report.py. Pre-parsing once at load time eliminates redundant work and provides structured data for all downstream consumers. Note: finalize_excel_file operates on openpyxl worksheet objects (not DataFrames), so it cannot use the _GT_PARSED cache directly -- the optimization there is moving regex compilation to module level.

Output: GT cache in dataframe_optimizer.py, updated consumers to use cache where possible (module-level GT_PATTERN elsewhere), cache cleanup before output, tests.
</objective>
